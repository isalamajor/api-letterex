# QUICK REFERENCE - 5 Minutos Antes de Entrevista

## 🔑 TUS FORTALEZAS (MENCIÓNALAS)

1. **JWT + Cookies Seguras**
   - httpOnly (previene XSS)
   - SameSite (previene CSRF)
   - 30 días expiración

2. **Validación Multi-nivel**
   - Verificación por código temporal
   - Encriptación con bcrypt
   - Upsert automático

3. **MongoDB Aggregation**
   - Conteo por idioma eficiente
   - Pipeline con $match y $group
   - Mejor que N queries

4. **Manejo de Archivos Seguro**
   - Validación dual (extensión + mimetype)
   - Limpieza automática de antiguos
   - 5MB límite

5. **Sugerencias Inteligentes**
   - Usuarios que enseñan tu idioma de aprendizaje
   - Usuarios a los que les enseñas
   - Random fill si hay espacios

---

## ⚠️ DEBILIDADES (ESTAR PREPARADO)

1. **Falta Rate Limiting**
   - Brute force en login
   - Solución: express-rate-limit

2. **Sin Logging/Auditoría**
   - No se registran acciones sensibles
   - Solución: Modelo Audit

3. **Validación Ligera de Email**
   - Solo check if (!email)
   - Solución: usar librería validator

4. **N+1 Queries en getFriends**
   - Promise.all hace query por usuario
   - Solución: $lookup aggregation

5. **Soft Delete sin implementar**
   - Campo existe pero no se usa
   - Solución: agregar deleted: true en querys

## 📋 RESPUESTAS RÁPIDAS

**"¿Cómo autentificas?"**
→ JWT en cookie httpOnly, validación en middleware, 10 días expiration

**"¿Cómo mantienes seguras las contraseñas?"**
→ bcrypt con salt rounds, comparación en tiempo constante

**"¿Dónde ves mayor oportunidad de mejora?"**
→ Rate limiting para prevenir brute force, RBAC para permisos, auditoría de acciones

**"¿Cómo escalarías para 1M usuarios?"**
→ Índices, caching con Redis, queue async para emails, microservicios

**"¿Algún bug encontrado?"**
→ Validación de email podría ser más estricta, falta rate limiting
